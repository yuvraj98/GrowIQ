Chapter 1
Executive Summary & Business Problem
GrowIQ is a SaaS platform that gives digital marketing agencies and business owners a single
command center to track campaigns, manage clients, generate AI-powered insights, and automate
invoicing — replacing 6+ disconnected tools with one intelligent platform.
1.1 The Problem
Indian SMBs spend ■50,000–■2,00,000/month on digital marketing yet have no unified way to measure
ROI across channels. Agencies managing 10+ clients juggle Meta Ads Manager, Google Ads, Analytics,
spreadsheets, WhatsApp, and accounting tools separately, leading to missed optimisations, delayed
reporting, and client churn.
 No single dashboard — data lives in 5–7 disconnected tools
 Manual reporting consumes 15–20 hours/week per agency
 No AI to detect underperforming campaigns in real-time
 Client invoicing done manually via Excel or Tally — error-prone
 No revenue forecasting or business health scoring
 Agencies lose clients because they cannot prove ROI clearly
 Small businesses cannot afford enterprise tools like HubSpot or Salesforce
1.2 The Solution — GrowIQ
GrowIQ solves every one of these problems in a single subscription. It auto-connects to Meta Ads,
Google Ads, GA4, and SEO tools via APIs, processes all data using the Claude AI engine, and delivers
actionable insights, automated reports, and branded client dashboards — all from one login.

System Design — High Level (HLD)
5.1 Architecture Overview
GrowIQ follows a three-tier architecture: a Next.js frontend served via Vercel CDN, a Node.js/Express
REST API backend running on AWS EC2, and a PostgreSQL database with Redis caching layer. A
separate AI microservice handles all Claude API calls asynchronously via a job queue.

Chapter 11
Security & Infrastructure
11.1 Security Measures
Authentication
JWT access tokens (15 min expiry) + refresh tokens (7 days).
Stored in httpOnly cookies.
Password Storage
bcrypt with salt rounds = 12. Never stored in plain text.
OAuth Token Storage
Meta/Google tokens encrypted at rest using AES-256 before
DB storage.
API Rate Limiting
express-rate-limit: 100 req/min per user. Prevents brute force
and API abuse.
Input Validation
Joi schema validation on all incoming request bodies. Prevents
injection attacks.
SQL Injection
Parameterised queries via pg library. No raw string
concatenation in SQL.
CORS Policy
Whitelist of allowed origins. Reject all others.
HTTPS Enforcement
SSL/TLS on all endpoints. HSTS header. HTTP to HTTPS
redirect.
Secrets Management
All API keys in .env. Never committed to git. AWS Secrets
Manager in production.
Client Data Isolation
All queries filter by agency_id. Clients can only see their own
data.
Audit Logging
All data modifications logged to audit_logs table with user ID
and timestamp.
Webhook Verification
Razorpay webhook signature verified using HMAC-SHA256
before processing.
